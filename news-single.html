JetBrains выпустила обновлённый менеджер Toolbox 2.0

4 часа назад

Прошлая версия Toolbox устанавливалась в пользовательском каталоге с постоянно меняющимся путём. Теперь каталог установки по умолчанию соответствует рекомендациям операционной системы. Toolbox 2.0 использует следующие пути:

  • %LocalAppData%\Programs для Windows;
  • ~/Applications для macOS;
  • ~/.local/share/JetBrains/Toolbox/apps для Linux.

Также разработчики отметили, что пока Toolbox может устанавливать приложения только в пользовательские каталоги. В будущем это планируют исправить путём добавления возможности повышать разрешения для утилиты. Тогда будет возможна установка в общесистемные каталоги.

В прошлой версии Toolbox для каждого обновления инструмента создавалась папка с номером версии в качестве названия. Из-за этого каждая новая версия имела новый путь к файловой системе. Теперь обновление устанавливается в текущую папку, сохраняя постоянный путь для инструмента. При этом сохранили возможность устанавливать рядом несколько версий одного инструмента.

Загрузка, распаковка и прочие операции обновления выполняются в фоновом режиме, давая пользователю возможность продолжать работать с обновляемым приложением. После завершения подготовительного этапа появится предупреждение о том, что необходима перезагрузка для копирования новых файлов. Обновление применится при ближайшем закрытии инструмента.

В Windows все установленные приложения JetBrains отображаются в соответствующем разделе настроек. Ярлыки после обновления не аннулируются, а брандмауэр Windows запоминает список разрешений.
